# Android Studio Build

This project is an Expo / React Native app with a generated native Android project.

## Open In Android Studio

Open this folder:

```txt
/Users/yutong/Documents/Reading_APP/android
```

Do not open only `android/app`.

## Build A Test APK

In Android Studio:

1. Wait for Gradle sync to finish.
2. Open `Build`.
3. Choose `Build Bundle(s) / APK(s)`.
4. Choose `Build APK(s)`.

The APK is usually generated here:

```txt
android/app/build/outputs/apk/debug/app-debug.apk
```

Debug APKs are mainly for development. If you install `app-debug.apk` directly on a phone, it may need the Metro development server running on your computer.

## Build A Release APK

For local testing, Android Studio can build the existing release variant.

The standalone APK generated by the local release build is here:

```txt
android/app/build/outputs/apk/release/app-release.apk
```

For Play Store or public distribution, create a real release signing key first. The generated project currently uses the debug key for release builds, which is okay for testing but not for production publishing.

## API Domain

The app uses this production API by default:

```txt
https://reading-app-sigma.vercel.app
```

After binding the final domain, update:

```txt
EXPO_PUBLIC_API_URL=https://your-final-domain
```
