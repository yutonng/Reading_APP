## 1. 软件概述

轻轻读移动阅读软件是一款移动端电子书阅读应用，主要用于书籍内容展示、逐页阅读、阅读进度记录和后台内容维护。用户可在首页查看书籍列表，进入阅读页后通过点击或滑动方式翻页阅读；管理员可在后台录入书籍标题、作者、简介和正文内容，系统自动将正文拆分为阅读页面。

| 项目 | 要求 |

## 3. 首页功能

用户打开软件后进入首页。首页显示当前可阅读的书籍列表，每条书籍信息包括书名、作者、页数、阅读状态和简介。

如果用户已经阅读过某本书，首页会显示上次阅读信息，包括书名、作者和上次阅读页码。用户点击“继续”按钮可直接进入上次阅读位置。

当远程内容加载失败时，首页会显示离线提示，并展示软件内置的离线书籍内容，保证用户仍可继续体验基础阅读功能。

## 4. 书籍阅读

用户在首页点击任意书籍后进入阅读页。阅读页顶部显示当前书名，下方显示阅读进度条，中间区域显示当前页正文，底部显示当前页码和总页数。

阅读页支持以下操作：

- 点击屏幕右侧区域进入下一页。
- 点击屏幕左侧区域返回上一页。
- 向左滑动进入下一页。
- 向右滑动返回上一页。
- 通过顶部返回按钮回到首页。

用户翻页时，系统会显示简短的翻页视觉效果，并自动记录当前阅读页码。

## 5. 阅读进度

软件会为每本书保存独立的阅读进度。用户再次进入同一本书时，系统会优先恢复到上次阅读的位置。首页的“上次阅读”区域会根据最近更新时间显示最近阅读的书籍。

当用户读到最后一页时，软件会显示“已完结”状态，并提供返回首页继续选择书籍的入口。

## 6. 后台内容管理

管理员可进入后台上传页面维护书籍内容。后台页面包含书名、作者、一句话简介和正文内容输入区域。

正文内容以空行作为分页依据。管理员每输入一段正文并用空行分隔，系统会将其识别为一个阅读页面，并在页面中显示当前页数统计。

管理员填写完整信息后点击“保存”按钮，系统将书籍内容提交到服务端保存。保存完成后，用户返回首页刷新即可查看新书籍。

## 7. 管理员接口

后台管理功能依赖管理员登录和服务端接口。系统提供管理员会话校验、登录、退出登录、书籍新增、书籍修改、书籍删除和书籍列表读取等能力。未登录管理员访问写入接口时，系统会拒绝操作并提示需要登录后台。

## 8. 异常提示

软件在以下情况下会显示提示信息：

- 书籍内容加载失败时，首页提示当前显示离线内容。
- 后台保存失败时，页面显示失败原因。
- 访问不存在的书籍时，阅读页提示没有找到对应书籍，并提供返回选书入口。
- 管理员未登录或登录信息错误时，接口返回相应错误提示。

## 9. 数据说明

软件主要处理以下数据：

- 书籍标题。
- 作者名称。
- 书籍简介。
- 正文分页内容。
- 创建时间和更新时间。
- 用户阅读页码和进度更新时间。
- 管理员登录会话。

用户阅读进度保存在本地设备中，书籍内容可从服务端读取；当服务端不可用时，软件使用内置书籍数据作为兜底内容。
