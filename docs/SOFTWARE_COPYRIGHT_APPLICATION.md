## 软件功能简介草稿

轻轻读移动阅读软件是一款面向移动端用户的电子书阅读应用。软件支持从服务端获取书籍列表，展示书名、作者、简介和阅读页数，并记录用户的最近阅读位置。用户可进入阅读页进行逐页阅读，支持点击左右区域翻页、横向滑动翻页、阅读进度条显示、页码显示、阅读完成提示和继续阅读入口。软件在网络内容加载失败时可显示本地离线内容，保障基础阅读体验。

软件同时提供后台内容管理能力，管理员可录入书名、作者、简介和正文内容，系统会将正文按段落解析为阅读页，并通过接口保存、更新或删除书籍数据。后台接口包含管理员登录、会话校验、退出登录、书籍读取和书籍写入等功能。整体系统采用移动端界面、共享核心逻辑、接口服务和数据存储分层设计，便于后续同时支持 H5、Android 和 iOS 版本。

## 主要功能点

- 书籍列表展示：显示书名、作者、简介、页数和阅读状态。
- 最近阅读：根据本地阅读进度定位上次阅读书籍和页码。
- 阅读器：支持逐页阅读、点击翻页、滑动翻页、翻页动画和阅读完成提示。
- 阅读进度：自动保存每本书的阅读页码和更新时间。
- 离线兜底：远程接口不可用时使用本地内置书籍数据。
- 内容管理：后台录入书籍标题、作者、简介和正文内容。
- 内容分页：按空行将正文解析为多个阅读页面。
- 管理接口：支持书籍新增、修改、删除、查询。
- 管理员鉴权：支持登录、退出和后台接口访问控制。
- 多端架构：前端页面、核心逻辑、服务接口和数据模型分层。

- 客户端系统：Android。

## 本项目建议提交的源程序范围

本项目核心源代码超过 60 页，可按常规方式提交源程序前 30 页和后 30 页。建议优先覆盖以下文件中的代码：

- `app/index.tsx`：书籍列表、最近阅读、离线提示。
- `app/reader/[id].tsx`：阅读器、翻页、滑动、进度展示。
- `app/admin.tsx`：后台上传页面。
- `src/core/reading-progress.ts`：阅读进度核心逻辑。
- `src/core/book-content.ts`：正文分页和书籍草稿转换逻辑。
- `src/services/books.ts`：书籍接口客户端、登录和管理 API 调用。
- `src/services/reading-progress.ts`：本地阅读进度存储。
- `server/core.mjs`：服务端接口、管理员鉴权、书籍读写。
- `api/books/index.mjs`、`api/books/[id].mjs`：书籍 API 入口。
- `api/auth/login.mjs`：登录 API 入口。

注意：提交源码材料时应尽量使用自主编写的业务代码，不建议把 `node_modules`、构建产物、系统生成文件、第三方库源码作为主要材料。

## 上架中国安卓应用市场的额外提醒

- 软著证书上的著作权人应尽量与应用市场开发者主体一致。本项目当前按个人著作权人“于童”准备。
- 软著软件名称应尽量与 App 上架中文名一致或可明显对应。当前建议登记为“轻轻读移动阅读软件”。
- 包名 `com.yutong.readingapp` 与证书通常不会完全绑定，但市场审核可能要求 App 名、开发者主体和软著名称能互相解释。
- 若用个人软著给公司主体上架，或公司软著给其他主体上架，可能需要额外授权书。
- 如涉及书籍内容上架，除软件著作权外，还要注意内容版权、出版物/网络文化/信息服务等合规要求，具体取决于实际内容来源和市场要求。
